# 習慣追蹤器 (Habit Tracker)

一個功能完整、界面美觀的習慣追蹤應用，幫助您建立和維持良好的習慣。使用純 HTML/CSS/JavaScript 開發，無需任何框架或後端服務。

![Status](https://img.shields.io/badge/Status-Active-success)

## 功能特色

### 核心功能
- ✅ **習慣管理**：新增、編輯、刪除習慣
- 📅 **每日打卡**：簡單點擊即可完成每日打卡
- 🔥 **連續追蹤**：自動計算當前連續天數和最長連續記錄
- 📊 **統計分析**：即時顯示總習慣數、今日完成數、完成率等
- 🎨 **多彩分類**：支援健康、學習、工作、娛樂等多種類別
- 🌈 **自訂顏色**：為每個習慣選擇專屬顏色

### 進階功能
- 🗓️ **熱力圖**：類似 GitHub 貢獻圖的年度打卡熱力圖（365天）
- 📈 **趨勢圖表**：最近30天的完成趨勢視覺化
- 💾 **本地儲存**：所有資料儲存在瀏覽器本地，無需註冊登入
- 📤 **資料匯出**：將習慣資料匯出為 JSON 檔案
- 📥 **資料匯入**：從備份檔案還原資料
- 📱 **響應式設計**：完美支援桌面、平板、手機等各種裝置

## 快速開始

### 安裝方式

1. **直接使用**
   - 下載所有檔案到本地資料夾
   - 用瀏覽器開啟 `index.html` 即可使用
   - 無需安裝任何依賴或執行建置步驟

2. **透過本地伺服器**
   ```bash
   # 使用 Python (推薦用於開發)
   python -m http.server 8000
   # 或使用 Node.js
   npx serve
   ```
   然後在瀏覽器開啟 `http://localhost:8000`

### 瀏覽器要求
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

## 使用指南

### 1. 新增習慣

1. 點擊右上角的「**新增習慣**」按鈕
2. 填寫習慣資訊：
   - **習慣名稱**（必填）：例如「每日運動30分鐘」
   - **描述**（選填）：描述這個習慣的目標和意義
   - **類別**（必填）：選擇健康、學習、工作、娛樂或其他
   - **顏色**（必填）：選擇一個代表顏色，方便辨識
3. 點擊「**儲存**」完成新增

### 2. 每日打卡

- 在習慣卡片底部找到「**今日打卡**」按鈕
- 點擊按鈕完成打卡，按鈕會變成綠色「已完成」
- 再次點擊可以取消打卡
- 打卡後會自動更新連續天數和統計資料

### 3. 查看詳細資訊

點擊習慣卡片的標題區域，會開啟詳細資訊視窗，包含：

#### 統計數據
- **當前連續**：最近連續打卡的天數
- **最長連續**：歷史最長連續打卡記錄
- **總完成次數**：累計打卡次數
- **完成率**：自建立以來的完成百分比

#### 視覺化圖表
- **打卡熱力圖**：365天的打卡記錄，顏色越深代表打卡越頻繁
- **完成趨勢**：最近30天的打卡趨勢圖

### 4. 編輯習慣

1. 點擊習慣卡片右上角的「✏️ 編輯」按鈕
2. 修改習慣資訊
3. 點擊「**儲存**」確認變更

### 5. 刪除習慣

1. 點擊習慣卡片右上角的「🗑️ 刪除」按鈕
2. 確認刪除操作
3. **注意**：刪除操作無法復原，建議先匯出資料備份

### 6. 篩選習慣

使用頂部的分類標籤快速篩選不同類別的習慣：
- **全部**：顯示所有習慣
- **健康**：運動、飲食、睡眠等
- **學習**：閱讀、學習新技能等
- **工作**：工作相關的習慣
- **娛樂**：興趣愛好相關
- **其他**：其他類別

### 7. 資料備份與還原

#### 匯出資料
1. 點擊右上角的「**匯出**」按鈕
2. 系統會自動下載一個 JSON 檔案（檔名包含匯出日期）
3. 妥善保存此檔案作為備份

#### 匯入資料
1. 點擊右上角的「**匯入**」按鈕
2. 選擇之前匯出的 JSON 檔案
3. 確認匯入操作
4. **注意**：匯入會覆蓋現有資料，建議先匯出當前資料

## 統計說明

### 儀表板統計

應用頂部的統計卡片提供整體概覽：

- **總習慣數**：目前追蹤的習慣總數
- **今日完成**：今天已經打卡的習慣數量
- **最長連續**：所有習慣中最長的連續打卡天數
- **完成率**：整體平均完成率

### 連續天數計算

- **當前連續**：從今天或昨天開始，往前連續打卡的天數
  - 如果今天已打卡，從今天開始計算
  - 如果今天未打卡但昨天有，從昨天開始計算
  - 中斷一天即重新計算

- **最長連續**：歷史上連續打卡天數的最高記錄

### 完成率計算

完成率 = (總打卡次數 ÷ 自建立以來的總天數) × 100%

例如：
- 習慣建立了 10 天
- 打卡了 8 次
- 完成率 = (8 ÷ 10) × 100% = 80%

## 資料儲存

### 本地儲存機制

- 所有資料儲存在瀏覽器的 **localStorage** 中
- 無需網路連線，完全離線可用
- 資料僅存在於當前裝置和瀏覽器
- 清除瀏覽器資料會導致習慣記錄遺失

### 資料結構

```json
{
  "habits": [
    {
      "id": "唯一識別碼",
      "name": "習慣名稱",
      "description": "習慣描述",
      "category": "類別",
      "color": "顏色代碼",
      "createdAt": "建立時間",
      "checkins": ["2025-01-01", "2025-01-02"],
      "currentStreak": 2,
      "longestStreak": 5
    }
  ]
}
```

### 備份建議

1. **定期匯出**：建議每週或每月匯出一次資料
2. **多地備份**：將匯出的 JSON 檔案儲存在雲端硬碟
3. **換裝置時**：先匯出資料，再於新裝置匯入
4. **瀏覽器更新前**：重大更新前先備份資料

### 核心類別與方法

#### HabitTracker 類別

主要的應用程式控制器，負責管理所有習慣追蹤功能。

**資料管理**
- `loadHabits()` - 從 localStorage 載入習慣資料
- `saveHabits()` - 儲存習慣資料到 localStorage
- `generateId()` - 生成唯一的習慣 ID

**CRUD 操作**
- `addHabit(habitData)` - 新增習慣
- `updateHabit(id, habitData)` - 更新習慣
- `deleteHabit(id)` - 刪除習慣

**打卡管理**
- `toggleCheckin(id)` - 切換打卡狀態
- `isCheckedToday(habit)` - 檢查今天是否已打卡
- `updateHabitStreaks(habit)` - 更新連續天數

**統計計算**
- `calculateCompletionRate(habit)` - 計算完成率
- `updateStats()` - 更新整體統計

**視覺化**
- `renderHeatmap(habit)` - 渲染熱力圖
- `renderTrendChart(habit)` - 渲染趨勢圖

**資料匯入匯出**
- `exportData()` - 匯出資料為 JSON
- `importData()` - 匯入 JSON 資料

### 效能優化

1. **事件委派**：使用事件委派減少事件監聽器數量
2. **本地計算**：所有統計在前端計算，無需伺服器
3. **漸進增強**：基礎功能優先，視覺化功能漸進增強

## 常見問題

### Q: 資料會同步到其他裝置嗎？
A: 不會。資料儲存在本地瀏覽器，如需在其他裝置使用，請使用匯出/匯入功能。

### Q: 清除瀏覽器快取會影響資料嗎？
A: 是的。清除瀏覽器資料會刪除 localStorage，建議定期匯出備份。

### Q: 可以追蹤多少個習慣？
A: 理論上沒有限制，但建議保持在 10-20 個以內，更容易管理和維持。

### Q: 連續天數是如何計算的？
A: 從今天或昨天開始往前連續打卡的天數。如果中斷超過一天，會重新計算。

### Q: 可以修改過去的打卡記錄嗎？
A: 目前版本僅支援當日打卡/取消打卡。如需修改歷史記錄，可以：
1. 匯出資料
2. 編輯 JSON 檔案中的 `checkins` 陣列
3. 重新匯入資料

### Q: 支援多人使用嗎？
A: 不支援。每個瀏覽器獨立儲存資料，如需多人使用，建議每人使用不同的瀏覽器設定檔。

### Q: 為什麼有些習慣顯示完成率超過100%？
A: 不會發生。完成率計算方式是「打卡次數 ÷ 自建立以來的天數」，最高為 100%。

## 最佳實踐

### 建立有效的習慣

1. **從小處開始**：選擇容易達成的小習慣
2. **具體明確**：「每天運動30分鐘」比「多運動」更好
3. **數量適中**：建議同時追蹤 3-7 個習慣
4. **保持彈性**：偶爾中斷不要氣餒，重新開始即可

### 使用技巧

1. **每天固定時間檢查**：例如早上或睡前
2. **設定瀏覽器書籤**：方便快速開啟
3. **定期檢視統計**：每週查看熱力圖和趨勢
4. **及時調整**：無法維持的習慣要適時調整或刪除

### 資料管理

1. **每月備份一次**：月初匯出上個月的資料
2. **年度回顧**：年底查看熱力圖，回顧一年的進步
3. **定期清理**：刪除已經不需要追蹤的習慣
