## Quick Start

### Option 1: Local File (No Server Required)

Simply open `index.html` in your web browser:

```bash
# Open with default browser
xdg-open index.html

# Or use a specific browser
firefox index.html
google-chrome index.html
```

### Option 2: Local Web Server

For better experience, serve with a local web server:

```bash
# Python 3
python3 -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000

# Node.js (if you have http-server installed)
npx http-server -p 8000
```

Then open `http://localhost:8000` in your browser.

## Usage

### 1. Load Boot Profile Data

**Method A: Upload JSON file**
- Click "Load Boot Profile" button
- Select a JSON file generated by backend tools
- Data will be visualized automatically

**Method B: Use Sample Data**
- Click "Load Sample Data" button
- Explore the interface with demo data

### 2. Explore Visualizations

**Timeline View**
- See the entire boot sequence at a glance
- Each stage is color-coded and sized by duration
- Hover over stages for details

**Boot Stages**
- Detailed breakdown of each boot phase
- Progress bars show relative time consumption
- Table view for precise numbers

**Services**
- Top 10 slowest services ranked
- Identify bottlenecks in service initialization
- Detailed timing information

**AI Recommendations**
- Smart optimization suggestions
- Severity-coded (Critical/High/Medium/Low)
- AI insights for each recommendation
- Actionable suggestions

### 3. Export Reports

Use browser's print function to save reports:
- `Ctrl+P` (Windows/Linux) or `Cmd+P` (Mac)
- Save as PDF for documentation

## Supported Data Formats

The frontend accepts JSON files with the following structure:

```json
{
  "timestamp": "2025-01-01T00:00:00",
  "total_time_ms": 8452,
  "stages": [
    {
      "name": "Firmware",
      "time_ms": 2100
    },
    {
      "name": "Kernel",
      "time_ms": 2800
    }
  ],
  "slow_services": [
    {
      "name": "NetworkManager.service",
      "time_ms": 2341,
      "active_ms": 2341
    }
  ],
  "recommendations": [
    {
      "type": "firmware",
      "severity": "high",
      "issue": "Firmware initialization takes 2.10s",
      "suggestion": "Enable Fast Boot in BIOS/UEFI settings",
      "ai_insight": "Modern UEFI firmware can skip POST checks."
    }
  ]
}
```

## Integration with Backend Tools

### SystemD Analyzer

```bash
# Run analyzer
python3 ../backend/profilers/systemd-analyzer.py

# Load generated file
# -> systemd_analysis.json
```

### C Profiler

```bash

# Load generated file
# -> boot_profile.json
```

### Bottleneck Detector

```bash

python3 ../backend/analyzers/bottleneck-detector.py systemd_analysis.json

# Load generated file
# -> bottleneck_analysis.json
```

## Troubleshooting

### JSON File Won't Load

- Check JSON syntax with a validator
- Ensure file has correct structure
- Try "Load Sample Data" to verify frontend works

### Charts Not Displaying

- Check browser console for errors (F12)
- Ensure JavaScript is enabled
- Try a different browser

### Styling Issues

- Clear browser cache (Ctrl+Shift+R)
- Disable browser extensions
- Check CSS file is loaded
