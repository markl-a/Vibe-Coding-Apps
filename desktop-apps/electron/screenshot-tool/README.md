# 📸 Screenshot Tool - AI 智能截圖工具

> 🤖 **AI-Enhanced** | 功能強大的螢幕截圖工具，集成 AI 文字識別和圖片分析

輕量級但功能強大的跨平台螢幕截圖工具，支援全螢幕、區域和視窗截圖，並整合 AI 智能功能。

## ✨ 功能特色

### 基礎截圖功能
- 🖥️ **全螢幕截圖** - 捕獲整個螢幕
- 🎯 **區域截圖** - 自由選擇截圖區域
- 🪟 **視窗截圖** - 捕獲特定視窗
- 🖼️ **簡單編輯器** - 基本的圖片預覽和編輯
- 📋 **快速複製** - 自動複製到剪貼簿
- 💾 **自動儲存** - 自定義儲存位置和格式

### 🤖 AI 智能功能 (新!)

#### OCR 文字識別
- ✨ 使用 GPT-4 Vision 進行智能文字識別
- 🌐 支援多種語言
- 📝 自動格式化為 Markdown
- 📊 表格自動轉換

#### 圖片描述生成
- 🔍 AI 自動分析圖片內容
- 📖 生成詳細的圖片描述
- 🎨 識別場景、物體、顏色等元素

#### 智能翻譯
- 🌍 將識別的文字翻譯成多種語言
- 🇺🇸 🇯🇵 🇰🇷 支援英文、日文、韓文等
- 📄 保持原文格式

### 其他功能
- ⚡ **全域快捷鍵** - 隨時隨地快速截圖
- 📚 **歷史記錄** - 保存最近 20 張截圖
- 🎨 **主題切換** - 深色/淺色主題
- 🔔 **系統托盤** - 最小化到托盤運行
- ⚙️ **自定義設定** - 豐富的配置選項

## 🚀 快速開始

### 安裝依賴

```bash
cd screenshot-tool
npm install
```

### 設定 AI 功能（可選）

#### 方法 1: 使用環境變數
```bash
export OPENAI_API_KEY="your-api-key-here"
```

#### 方法 2: 使用配置檔案
1. 複製 AI 配置檔案：
```bash
cp ../shared/ai-config.example.js ../shared/ai-config.js
```

2. 編輯 `../shared/ai-config.js` 並填入你的 OpenAI API Key：
```javascript
module.exports = {
  OPENAI_API_KEY: 'your-api-key-here',
  // ...其他設定
};
```

**注意**: 沒有 AI 功能的情況下，應用的基本截圖功能仍可正常使用。

### 啟動應用

```bash
npm start
```

## ⌨️ 快捷鍵

### 全域快捷鍵（系統級）
- `Ctrl/Cmd + Shift + A` - 全螢幕截圖
- `Ctrl/Cmd + Shift + S` - 區域截圖
- `Ctrl/Cmd + Shift + W` - 視窗截圖

### 編輯器快捷鍵
- `Ctrl/Cmd + S` - 儲存截圖
- `Ctrl/Cmd + C` - 複製到剪貼簿
- `ESC` - 關閉編輯器

## 📖 使用說明

### 基本截圖流程

1. **選擇截圖模式**
   - 點擊主介面的快捷按鈕
   - 或使用全域快捷鍵

2. **編輯截圖**（可選）
   - 截圖會在編輯器中開啟
   - 可以使用 AI 功能分析圖片

3. **儲存或複製**
   - 點擊「儲存」按鈕保存到本地
   - 點擊「複製」按鈕複製到剪貼簿

### 使用 AI 功能

#### OCR 文字識別

1. 在編輯器中點擊「🤖 OCR 識別」按鈕
2. 等待 AI 處理（通常需要 5-10 秒）
3. 識別結果會顯示在右側面板
4. 可以複製結果或進行翻譯

**使用場景**:
- 📄 識別截圖中的文字內容
- 📊 將圖片中的表格轉換為 Markdown
- 📖 提取文檔圖片中的文字
- 🌐 識別外語文字並翻譯

#### 圖片描述生成

1. 在編輯器中點擊「📝 圖片描述」按鈕
2. AI 會分析圖片內容
3. 生成詳細的圖片描述

**使用場景**:
- ♿ 為視障用戶生成圖片描述
- 🔍 快速了解圖片內容
- 📝 生成圖片的文字說明
- 📚 用於文檔註解

#### 智能翻譯

1. 先使用 OCR 識別文字
2. 點擊「🌐 翻譯」按鈕
3. 輸入目標語言代碼（如 en, ja, ko）
4. 查看翻譯結果

## ⚙️ 設定選項

### 儲存設定
- **儲存位置** - 自定義截圖儲存目錄
- **檔案格式** - PNG、JPEG、WebP
- **檔案名稱** - 支援時間占位符
  - `%Y` - 年份（4位）
  - `%m` - 月份（2位）
  - `%d` - 日期（2位）
  - `%H` - 小時（24小時制）
  - `%M` - 分鐘
  - `%S` - 秒

### 行為設定
- **自動複製到剪貼簿** - 截圖後自動複製
- **顯示通知** - 截圖成功後顯示系統通知
- **截圖顯示游標** - 是否在截圖中顯示滑鼠游標

### 品質設定
- **圖片品質** - 1-100，影響檔案大小

## 💡 使用技巧

### 1. 快速 OCR 工作流程

```
1. Ctrl+Shift+S 開始區域截圖
2. 選擇包含文字的區域
3. 截圖自動在編輯器中開啟
4. 點擊「OCR 識別」
5. 複製識別結果
```

### 2. 多語言文檔處理

```
1. 截圖外語文檔
2. 使用 OCR 識別文字
3. 點擊「翻譯」翻譯成中文
4. 複製翻譯結果
```

### 3. 圖片內容分析

```
1. 截圖需要分析的圖片
2. 點擊「圖片描述」
3. AI 會告訴你圖片包含什麼內容
```

### 4. 批量處理歷史截圖

```
1. 在主介面查看「最近截圖」
2. 點擊任一歷史截圖重新開啟
3. 使用 AI 功能進行分析
```

## 🔒 隱私和安全

### 本地優先
- ✅ 所有截圖都儲存在本地
- ✅ 歷史記錄僅保存在本機
- ✅ 不會上傳任何資料到雲端（除 AI 功能外）

### AI 功能隱私
- ⚠️ AI 功能需要將圖片發送到 OpenAI API
- 🔒 僅在用戶主動點擊 AI 按鈕時才發送
- ✅ 可以完全不設定 API Key，不使用 AI 功能
- ⚠️ 建議不要對敏感資訊使用 AI 功能

### API Key 安全
- 🔐 API Key 儲存在本地加密存儲中
- ✅ 不會被記錄或分享
- ✅ 僅用於 AI 功能調用

## 🐛 故障排除

### AI 功能無法使用

**問題**: 點擊 AI 按鈕提示「請先設定 OpenAI API Key」

**解決方案**:
1. 確認已設定 API Key（環境變數或配置檔案）
2. 檢查 API Key 是否有效
3. 確認有足夠的 API 額度
4. 檢查網路連接

### OCR 識別不準確

**問題**: 文字識別結果不正確

**建議**:
1. 確保截圖清晰，文字可讀
2. 避免截圖包含過多背景雜訊
3. 嘗試調整截圖區域，只包含文字部分
4. 某些手寫文字可能識別率較低
5. 複雜的表格可能需要手動調整

### 截圖儲存失敗

**問題**: 無法儲存截圖

**解決方案**:
1. 檢查儲存目錄是否存在
2. 確認有寫入權限
3. 檢查磁碟空間是否足夠
4. 確認檔案名稱格式正確

### 全域快捷鍵衝突

**問題**: 快捷鍵無法使用

**解決方案**:
1. 檢查是否與其他應用衝突
2. 在設定中修改快捷鍵（未來版本）
3. 使用介面按鈕代替快捷鍵
4. macOS 需要授予輔助功能權限
